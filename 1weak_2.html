REVIEW

SSD와 M.2
둘다 컴퓨터 저장장치의 종류이지만 저장장치의 형식과 요구 사항에서 차이점이 있다.
SSD는 하드디스크에 비해 더 높은 속도와 성능을 제공하는 저장장치이다.
NAND 기반 플래시 메모리로 구성되어 있어, 전원이 차단되어도 데이터를 보전할 수 있다.
M.2는 SSD와 동일한 기술이 들어갔지만 다른 하드웨어 인터페이스를 사용한다.
PCIE 인터페이스를 사용해 컴퓨터에서 데이터를 읽고 쓰기 위해 사용된다.
PCIE 인터페이스는 기존 SSD의 SATA보다 빠른 전송 속도를 가진다.

WOL
네트워크를 통해 저전력 모드 또는 종료 상태로 있는 컴퓨터를 원격으로 켤 수 있는 기술이다.
LAN에서 직접 시그널을 보내서 컴퓨터를 켜는 방식으로 동작하며 원격이면서 동시에 전원을 관리할 때 사용한다.
WOL 기능을 사용하려면 대상 컴퓨터의 하드웨어와 BIOS가 WOL을 지원해야 하며, 운영체제에서 WOL 기능이 활성화되 있어야 한다.
WOL은 공급업체를 위해 원격 관리, 데이터 센터 내 시스템 온오프를 제어하거나 야간 백업 및 유지 보수 작업 등을 할 때 유용하다.
하지만 네트워크 보안 측면에서 올바른 방식으로 적용하지 않으면 문제가 발생할 우려가 있으므로 전송 매직 패킷에 대한 액세스를 제어하는 것이 중요하다.

AC부팅
파워 서플라이와 메인보드 간의 상호작용에 의해 컴퓨터가 정전과 같은 상황에서 자동으로 부팅되도록 하는 기능이다.
이 기능이 구성되 있으면 전원 공급이 차단된 후 다시 복원되면 시스템이 자동으로 부팅한다.
컴퓨터 관리자가 원격으로 더 빠르게 시스템 작동을 복원하거나 사용 가능한 상태로 되돌릴 수 있기에 일부 비즈니스나 산업 시나리오에 유용하다.

에자일 기법
소프트웨어 개발이나 프로젝트 관리 방법론 중 하나.
이 기법은 주요 원칙들이 있다.

- 고객의 만족과 요구사항을 우선적으로 보고 변화에 빠르게 대응.

- 가치를 창출하는 기능과 요구사항에 중점을 두고 최소한의 비용, 시간으로 결과물 생산.

- 프로젝트 참여자들 간의 지속적이고 효과적인 의사소통을 강조.

- 유연한 계획을 세우고, 조정하며 프로젝트의 과정에 나타나는 변화에 대응.

폭포수 기법
이 기법도 위에 서술된 에자일 기법과 같은 방법론이며, 선형 단계를 따르는 과정을 이용한다.
각 단계가 순차적으로 진행되며 이전 단계가 완료된 후 다음 단계를 실행하는 것이 default이다.

1. 고객의 요구사항을 수집하고 문서화 한다.

2. 소프트웨어 시스템의 구조 및 구성 요소를 정의하며, 데이터 구조, 인터페이스 및 알고리즘과 같은 사항을 결정.

3. 소프트웨어를 실제로 개발하는 단계, 프로그래밍 언어를 사용해 코드를 작성하고 테스트.

4. 베타 버전을 임의로 정한 사용자 그룹에 배포해 테스트하며 오류, 버그 등을 식별하고 수정.

5. 마지막으로 수정된 버전을 실제 사용자들에게 배포해 사용자들이 사용.

6. 배포된 이후 발생할 수 있는 변경 요청, 추가 요구사항, 문제 해결을 위해 유지 보수.

HTTP
웹상에서 문서, 이미지, 동영상, 음성 등의 리소스들을 전송하기 위한 애플리케이션 프로토콜이다.
웹 브라우저, 웹 서버 간에 하이퍼텍스트 기반 정보를 교환할 때 사용한다.
HTTP는 클라이언트-서버 구조를 사용한다. 클라이언트는 HTTP 요청을 보내고 서버는 요청에 따른 HTTP 응답을 반환한다.
클라이언트와 서버 간에 요청, 응답을 교환한 후에는 연결을 끊는다.
이러므로써 서버가 많은 클라이언트의 요청을 처리할 수 있게 하며 리소스를 효율적으로 활용할 수 있다.

DHCP
네트워크에서 호스트한테 동적으로 IP주소, 관련 네트워크 구성 정보를 할당하는 프로토콜이다.
네트워크 관리자가 수동으로 IP주소를 할당하고, 기록할 필요가 없기 때문에 대규모 네트워크에서 IP 주소 관리를 단순화하고 효율높게 할 수 있다.
DHCP의 작동원리는 먼저, 클라이언트가 네트워크에 접속하면 DHCP 서버에 IP주소와 네트워크 구성 정보 요청을 한다.
서버는 클라이언트에게 임시 IP주소와 추가 네트워크 구성 정보를 제공한다.
클라이언트는 서버가 제공한 IP주소를 수락하고 네트워크 상에서 통신할 수 있게 된다.
IP주소가 만료되기 전에 클라이언트는 DHCP 서버에게 IP주소 연장 혹은 새로운 IP주소를 요청 가능하다.

DNS
인터넷에서 도메인 이름을 해당 IP 주소로 변환해주는 분산형 시스템.
인터넷에 연결된 디바이스들은 서로 고유한 IP주소를 가지고 있는데 이 주소를 사용하여 서로 통신한다.
하지만 사람들은 문자로 된 도메인 이름(ex: www.naver.com)을 기억하기 쉽다.
그래서 DNS가 도메인 이름과 IP주소를 매핑하여 웹 브라우저와 같은 클라이언트 애플리케이션에서 특정 웹사이트에 접근하는 데 사용한다.

OSI
통신 네트워크에서 데이터 전송에 관여하는 프로토콜과 서비스를 기능별로 구반하고 계층으로 나눈 참조 모델.
80년대 초기에 ISO에서 개발하였다.
이 모델은 총 7계층으로 이루어져 있고 각 계층은 특정한 기능들을 수행한다.
종류는 다음과 같다.

1 layer 물리 계층(physical layer):
주로 전기적, 기계적, 기능적인 특성을 이용해 통신 케이블로 데이터을 전송한다.
이 계층에서 사용되는 통신 단위는 비트이며 이것은 1과 0으로 나타내어진다.
단지 데이터을 전달만 할뿐 전송하려고 하는 데이터나 받으려고 하는 데이터가 무엇인지, 에러는 있는지 같은 경우는 신경 쓰지 않는다.
그저 데이터를 전기적인 신호로 변환해서 주고 받는 기능만 해준다.
여기에 속하는 장비는 통신 케이블, 리피터, 허브 등이 있다.

2 layer 데이터 링크 계층(data link layer):
위에 서술된 물리 계층을 통해 송수신 되는 정보의 오류와 흐름을 관리하여 안전한 정보의 전달을 수행할 수 있도록 도와준다.
필터 처럼 통신 할 때의 오류도 찾아주고 재전송도 하는 기능을 가지고 있다.
여기서는 MAC 주소를 가지고 통신하게 된다.
여기에서 전송되는 단위는 프레임이라고 한다.
브리지, 스위치 등이 이 계층에 속하는 장비이다.

3 layer 네트워크 계층(network layer):
여기에서 하는 가장 중요한 기능은 데이터를 목적지까지 가장 안전하고 빠르게 전달하는 기능이다.
이 기능을 라우팅이라고 하고 이 라우팅 하는 기술도 다양하다.
경로를 선택하고 주소를 정하고 경로에 따라 패킷을 전달해주는 것이 이 계층의 역할이다.
이 계층의 대표적인 장비는 라우터이고, 최근에는 앞서 서술한 데이터 링크 계층에 속하는 장비인 스위치에 라우팅 기능을 장착한 layer3 스위치도 있다.
여러개의 노드를 거칠 때마다 경로를 찾아주는 역할을 하는 계층이다.
다양한 길이의 데이터를 네트워크들을 통해 전달하고, 그 과정에서 전송 계층이 요구하는 서비스 품질을 제공하기 위한 기능적, 절차적 수단을 제공한다.
여기서 하는 작업은 라우팅, 흐름제어, 세그멘테이션, 오류 제어, 인터네트워킹등을 수행한다.
앞서 말했듯 라우터가 이 계층에서 동작하고 이 계층에서 동작하는 스위치도 있다.
데이터를 연결하는 다른 네트워크를 통해 전달함으로써 인터넷이 가능하게 만드는 계층이다.
논리적인 주소 구조(IP) 즉, 네트워크 관리자가 직접 주소를 할당하는 구조를 가지며, 계층적이다.
서브네트의 최상위 계층으로 경로를 설정하고, 청구 정보를 관리한다.
개방형 시스템들 사이에서 네트워크 연결을 설정, 유지, 해제하는 기능을 부여하고, 전송 계틍 사이에 네트워크 서비스 데이터 유닛을 교환하는 기능을 제공한다.

- IP 계층
TCP/IP 상에서 IP 계층이란 네트워크의 주소 즉, IP주소를 정의하고, IP 패킷의 전달 및 라우팅을 담당하는 계층이다.
OSI 7계층 모델의 관점에서 보았을 때 IP 계층은 네트워크 계층에 해당한다.
즉, 패킷을 목적지 까지 전달하는 역할 및 그에 수반되는 기타 역할을 한다.
IP 계층에서의 주요한 역할은 그 하위계층인 데이터링크 계층의 하드웨어적인 특성에 관계없이 독립적인 역할을 수행한다.
IP 계층 상에 있는 주요한 프로토콜은 IP, ICMP, 라우팅 프로토콜 등이 있다.

- IP 프로토콜
TCP/IP 기반의 인터넷 망을 통하여 데이터그램의 전달을 담당하는 프로토콜.
주요기능은 IP 계층에서 IP 패킷의 라우팅 대상이되고, IP 주소를 지정할 수 있다.
주요 특징은 다음과 같다.

* 에러 제어 및 흐름제어 기능이 전혀 없음. -> best-effort service

* 신뢰성을 확보하려면 IP 계층 위의 TCP와 같은 상위 트랜스포트 계층에 의존.

* 비연결성 데이터그램 방식으로 전달되는 프로토콜. -> connectionless

* 패킷의 완전한 전달을 보장하지 않음. -> unreliable

* IP 패킷 헤더 내 수신 및 발신 주소를 포함. -> IPv4 헤더, IPv6 헤더, IP 주소

* IP 헤더 내 바이트 전달 순서: 최상위 바이트를 먼저 보냄. -> big-endian

* 경우에 따라, 단편화가 필요함. -> IP 단편화 참조

* TCP, UDP, ICMP, IGMP 등이 IP 데이터그램에 실려서 전송.

4 layer 전송 계층(transport layer):
통신을 활성화하기 위한 계층이다.
보통 TCP 프로토콜을 이용하며, 포트를 열어서 응용프로그램들이 전송을 할 수 있게 한다.
만약 데이터가 왔다면 4계층에서 해당 데이터를 하나로 합쳐서 5계층에 던져 준다.
이 계층 까지는 물리적인 계층에 속한다.
전송 계층은 양 끝단의 사용자들이 신뢰성있는 데이터를 주고 받을 수 있도록 해 주어서, 상위 계층들이 데이터 전달의 유효성이나 효율성을 생각하지 않도록 해준다.
시퀸스 넘버 기반의 오류 제어 방식을 사용한다.
이 계층은 특정 연결의 유효성을 제어하고, 일부 프로토콜은 상태 개념이 있고, 연결 기반이다.
이는 전송 계층이 패킷들의 전송이 유효한지 확인하고 전송 실패한 패킷들을 다시 전송한다는 것을 뜻한다.
가장 잘 알려진 전송 계층의 예는 TCP이다.
종단간 통신을 다루는 최하위 계층으로 종단간 신뢰성 있고 효율적인 데이터를 전송하며, 기능은 오류검출 및 복구와 흐름제어, 중복검사 등을 수행한다.

- TCP 프로토콜
OSI 계층모델의 관점에서 전송 계층에 해당.
양종단 호스트 내 프로세스 상호 간에 신뢰적인 연결지향성 서비스를 제공.
IP의 비신뢰적인 최선형 서비스에다가 연결지향성 서비스를 제공.
신뢰적인 전송을 보장하기 때문에, 어플리케이션 구현이 한층 쉬워지게 됨.
패킷 손실, 중복, 순서바뀜 등이 없도록 보장.
TCP 하위계층인 IP 계층의 신뢰성 없는 서비스에 대해 다방면으로 신뢰성을 제공.

- UDP 프로토콜
전송 계층의 통신 프로토콜중 하나.
신뢰성이 낮은 프로토콜로써 완전성을 보증하진 않지만 가상회선을 굳이 확립할 필요가 없고 유연하며 효율적 응용의 데이터 전송에 사용.
비연결성이고 신뢰성이 없으며 순서화 되지 않은 데이터그램 서비스를 제공.
실시간 응용 밒 멀티캐스팅이 가능해서 빠른 요청과 응답이 필요한 실시간 응용에 적합하다.
헤더가 단순하기 때문에 헤더 처리에 많은 시간과 노력이 필요하지 않다.

5 layer 세션 계층(session layer):
데이터가 통신하기 위한 논리적인 연결을 말한다.
통신을 하기위한 대문같은 거라고 보면 된다.
하지만 4계층에서 연결을 맺고 종료하는 것이 가능하기 때문에 우리가 어느 계층에서 통신이 끊어졌나를 판단하는 것에 어려움이 있다.
그러므로 세션 계층은 4계층인 전송 계층과 무관하게 응용프로그램의 관점에서 봐야 한다.
기능은 세션 설정, 유지, 종료, 전송 중단시에 복구 등이 있다.
세션 계층은 양 끝단의 응용 프로세스가 통신을 관리하기 위한 방법을 제공한다.
동시 송수신 방식, 반이중 방식, 전이중 방식의 통신과 함께 체크 포인팅과 유휴, 종료, 다시 시작 과정등을 수행한다.
세션 계층은 TCP/IP 세션을 만들고 없애는 책임을 진다.

6 layer 표현 계층(presentation layer):
데이터 표현이 상이한 응용 프로세스의 독립성을 제공하고, 암호화 한다.
코드 간의 번역을 담당하여 사용자 시스템에서 데이터의 형식상 차이를 다루는 부담을 후술할 응용 계층으로부터 덜어준다.
MIME 인코딩이나 암호화 등의 동작이 이 계층에서 이루어진다.
EBCDIC로 인코딩된 문서 파일을 ASCII로 인코딩된 파일로 바꿔 주는 것이나 해당 데이터가 text인지, gif인지, jpg인지의 구분 등을 해주는 것이 표현 계층이다.

7 layer 응용 계층(application layer):
계층 구조의 최종 목적지로서 HTTP, FTP, SMTP, POP3, Telnet 등과 같은 프로토콜이 있다.
해당 통신 패킷들은 방금 나열한 프로토콜에 의해 모두 처리되고 우리가 사용하는 브라우저나, 메일 프로그램은 프로토콜을 쉽게 사용하게 해주는 응용프로그램이다.
모든 통신의 양 끝단은 HTTP와 같은 프로토콜이지 응용프로그램이 아니다.
이 계층은 응용 프로세스와 직접 관계하여 일반적인 응용 서비스를 수행한다.
일반적인 응용 서비스는 관련된 응용 프로세스들 사이의 전환을 제공한다.
예로, 가상 터미널, job transfer and manipulation protocol 등이 있다.

- HTTP 프로토콜
웹 상에서 웹 서버 및 웹 브라우저 상호 간의 데이터 전송을 위한 응용계층 프로토콜.
처음에는 www 상의 하이퍼텍스트 형태의 문서를 전달하는데 주로 이용했지만 지금은 이미지, 비디오, 음성 등 거의 모든 형식의 데이터 전송이 가능.

* 요청 및 응답의 구조이기 때문에 동작형태가 클라이언트 - 서버 모델로 동작한다.

* 메세지 교환 형태의 프로토콜이다.
클라이언트와 서버 간에 HTTP 메세지를 주고 받으며 통신하고 이는 SMTP 전자메일 프로토콜과 유사하다.

* HTTP의 응답 및 요청 메세지를 구성한다.

* 종단간 연결이 없고 이전의 상태를 유지하지 않는다.

Git과 Github
Git는 버전 관리 시스템중 하나로, 코드 변화를 관리하기 위한 도구이다.
Git은 소스 코드의 변화를 추적하고, 여러 명의 개발자들이 동시에 작업할 수 있도록 돕는 다양한 도구와 기능을 제공한다.
분산형 버전 관리 시스템이기 때문에, 다양한 브랜치를 통해 동시에 개발할 수 있고, 하나의 원격 저장소와 여러 개의 로컬 저장소로 구성된다.
Github는 Git이 개발되고 유기 관리되는 웹 기반 호스팅 서비스다.
Git 저장소를 호스팅하고, 개발자들이 협업할 수 있도록 다양한 기능을 제공해준다.
Github에는 공개 저장소와 비공개 저장소 둘 다 제공하며 소스코드, 이슈, pull 요청 등 다양한 정보관리가 가능하다.
그리고 Github로부터 다른 개발자들의 원격 저장소를 가져와 로컬 저장소와 병합하는 것 또한 가능하다.
이 둘을 쓰면서 얻을 수 있는 이점은 다음과 같다.

- 코드 변경 이력을 추적하고 관리할 수 있기 때문에 문제가 발생한 경우 언제든지 이전 상태로 돌아갈 수 있다.

- 다양한 브랜치를 통해 동시에 여러 기능을 개발할 수 있다.

- 여러 개발자들이 동시에 작업할 수 있으며 동시에 작업하더라도 충돌이 일어나지 않도록 협업이 가능하다.

- 코드 검토, 이슈 추적, 코드 리뷰, 커뮤니티 활동 등 다양한 기능을 제공한다.

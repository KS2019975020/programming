REVIEW

On-premise와 Off-premise
On-premise는 기업의 서버를 클라우드 같은 원격 환경에서 운영하는 방식이 아닌, 자체적으로 보유한 서버에 직접 설치해 운영하는 방식이다.
클라우드 컴퓨팅 기술이 나오기 전까지는 기업 인프라 구축의 일반적인 방식이다.
이 시스템의 장점은 기업의 비지니스 정보를 보안성 높게 관리할 수 있다는 것이고, 단점은 시스템을 구축하는데 있어서 많은 시간이 걸린다는 것과 많은 비용이 들어간다는 것이다.
보통 기업에서는 보안성 높은 데이터는 On-premise 시스템으로 관리한다.
Off-premise는 위에 서술한 클라우드 환경을 사용하는, 원격으로 관리할 수 있는 시스템이다.
On-permise는 기업내에서 구축하는 서버이지만 Off-premise는 클라우드 기반의 임대 서비스를 사용하냐의 차이가 있다.
Off-premise는 인터넷 네트워크에 연결된 서버팜이나 클라우드의 원격 실행환경에서 사용되는 소프트웨어를 뜻한다.

IDC
Internet Data Center의 준말.
서버 컴퓨터와 네트워크 회선 등을 제공하는 시설로, 서버를 한 데 모아 집중시킬 필요가 있을 때 설립한다.
인터넷 연결의 핵심이 되는 서버를 한 데 모아 집중시킬 필요가 있을 때 설립하는 시설을 말한다.
IDC를 통해 온라인 게임의 운영에 필요한 서버 컴퓨터와 네트워크 회선 등을 제공한다.
서버들을 한 데 모아놓았기 때문에 서버 호텔이라고도 부른다.
IDC는 서버들을 한 데 모아 안정적으로 운용해 주기 때문에 특히나 온라인 게임에서는 중요한 시설로 손꼽힌다.
예를 들어, 온라인 게임은 특성 상 서버가 없으면 서비스 자체가 힘들다.
그렇기 때문에 엄청나게 많은 서버가 필요한 RPG게임 같은 경우는 네트워크 속도와 관리 면의 이점 때문에 IDC를 이용하기도 한다.
IDC는 정전 등으로 인해 서버가 멈추면 안되기 때문에 절대 정전되지 않도록 설계한다.
혹시 모를 상황에 대비하기 위해 IDC는 전력 공급 업체로부터 우선적으로 전력을 공급받는 방법으로 여러 발전소에서 전력을 공급받을 수 있도록 설계한다.
IDC를 관리하는데는 인력이 필요하기도 하고 자리가 필요하기도 하다.
그렇기 때문에 대부분 통신사에서는 자체적으로 운영하는 IDC가 있으며 대기업들이 개별적으로 설립하여 사용하기도 한다.

IaaS
위에 서술된 Off-premise에 포함되는 서비스이다.
제 3사가 스토리지와 가상화와 같은 인프라 서비스를 인터넷을 통해 클라우드로 제공된다.
사용자는 운영체제 및 데이터, 애플리케이션, 미들웨어 및 런타임을 담당하고 제공업체는 사용자가 필요로 하는 네트워크, 서버, 가상화 및 스토리지의 관리와 액세스를 담당한다.
제공하는 업체가 사용자를 대신해 온사이트 데이터센터를 유지, 관리, 업데이트한다.
대신에, 사용자는 애플리케이션 프로그래밍 인터페이스(API) 또는 대시보드를 통해 인프라에 액세스하고 제어한다.
IaaS는 필요한 구성 요소만 구매하고 필요에 따라 확장 또는 축소할 수 있는 유연성을 제공한다.
간접비가 낮고 유지관리 비용이 들지 않기에 매우 경제적인 옵션이다.
개발 및 테스트 환경의 구축 및 제거가 빠르고 유연하다.
개발 환경에서 구축해야 할 인프라만 사용하고 필요에 따라 확장 또는 축소하며, 개발이 완료되면 사용을 중단하고 사용량에 대한 비용만 지불한다.
IaaS의 단점은 제공하는 업체의 보안 문제 가능성, 제공업체가 여러 클라이언트와 인프라 리소스를 공유해야 하는 멀티 테넌트 시스템 및 서비스 신뢰성이다.
사용자가 신뢰할 수 있는 제공업체를 처음부터 선정하는 것이 중요하다.
예로는 AWS, Microsoft Azure, Google Cloud 같은 퍼블릭 클라우드 공급업체들이 있다.

PaaS
IssA에서 조금 더 발전된 형태이다.
제공업체가 자체 인프라에서 하드웨어와 소프트웨어를 호스팅하고 이러한 플랫폼을 사용자에게 통합 솔루션, 솔루션 스택 또는 인터넷을 통한 서비스로 제공한다.
주로 개발자와 프로그래머에게 유용한 PaaS를 통해 사용자는 자체 애플리케이션을 개발, 실행, 관리할 수 있으며, 이 과정에서 일반적으로 이러한 프로세스와 관련된 인프라 또는 플랫폼을 구축하고 유지 관리할 필요가 없다.
이용자는 애플리케이션 코드를 작성, 빌드, 관리하지만 소프트웨어 업데이트 또는 하드웨어 유지관리와 같은 번거로움이 사라진다.
빌드 및 배포를 위한 환경이 사용자에게 제공된다.
개발자가 프레임워크를 개발하여 지속적으로 웹 기반 애플리케이션을 빌드 및 커스터마이징할 수 있는 플랫폼이다.
기본 소프트웨어 구성 요소를 활용하여 자체적인 애플리케이션을 개발하는 것도 가능하기 때문에 처음부터 일일히 코드를 작성해야 하는 수고를 조금 덜 수 있다.
예시로는 AWS Elastic Beanstalk, Heroku 가 있다.

SaaS
클라우드 애플리케이션 서비스라고도 하고, 이는 가장 포괄적인 형식의 클라우드 컴퓨팅 서비스이다.
모든 애플리케이션을 제공업체가 관리하고 웹 브라우저를 통해 제공된다.
제공업체가 소프트웨어 업데이트, 버그 수정 및 기타 일반 소프트웨어 유지관리 작업을 처리하며, 사용자는 대시보드 또는 API를 통해 애플리케이션에 연결한다.
개별적으로 시스템에 소프트웨어를 설치할 필요가 없고, 프로그램에 대한 그룹 액세스가 더욱 원활하고 안정적이다.
Gmail 같은 웹 기반 서비스가 지원되는 이메일 계정이 있다면 어디서든 컴퓨터에서 계정에 로그인하고 이메일을 수신할 수 있다.
이런 Gmail 같은 웹 기반 서비스가 SaaS의 형태라고 봐도 된다.
SaaS는 소프트웨어 설치 및 업데이트를 처리할 인력이나 대역폭이 없으며 최적화가 그다지 필요하지 않거나 주기적으로 사용되는 애플리케이션이 있는 소기업에 매우 유용한 옵션이다.
이는 시간과 유지관리를 줄일 수 있지만 제어, 보안 및 성능과 관련된 비용이 나가기 때문에 PaaS와 같이 신뢰 가능한 제공업체를 처음부터 선택하는 것이 중요하다.
예는 Dropbox, mega, Google Apps이 SaaS 형태라고 할 수 있다.

rm -rf와 위험성
rm 명령어는 파일을 지울 때 사용하는 명령어이다.
뒤에 -r 옵션을 붙이면 폴더를 지울 수 있게 되고, -f 옵션을 붙이면 강제로 지운다.
만약 위와 같이 rm -rf 뒤에 *을 붙이면 현재 가리키고 있는 디렉토리 내에 모든 파일을 지우기에 상당히 조심해서 사용해야 한다.
rm -rf . /* 만약 이런식으로 명령어를 쓴다면, 특히 root 계정으로 쓴다면 하드디스크 및 OS파일을 전부 지우기 때문에 더더욱 조심해야 한다.

grep
grep은 입력으로 전달된 파일의 내용에서 특정 문자열을 찾고자 할때 사용하는 명령어.
간단하게 문자열이 같은지 비교하면서 찾아주는 것이 아니라 정규 표현식에 의한 패턴 매칭 방식을 사용한다.

- 정규 표현식
특정 규칙을 가진 문자열 집합을 표현하기 위한 형식언어.
주로 문자열 패턴 매칭을 검사하거나 또는 문자열을 치환하기 위해 사용.
문자열 검색에 정규표현식을 적용하면 지정된 문자열의 문자가 단순히 같은지의 여부가 아닌 정규 표현식의 규칙에 매칭되는지를 검사한다.
예로 단순 문자열 검색에서 "*"은 문자 그대로 "*"을 의미하기 때문에 ("*" == "*")가 성립하지만 ("A" == "*")은 성립하지 않는다.
하지만 정규 표현식에서 "*"은 0개 이상의 모든 문자를 의미하므로, ("*" == "*")과 ("A" == "*") 모두 TRUE로 판단한다.

grep 명령어의 틀은 아래와 같다.
$ grep [option] [pattern] [file]
grep 명령어에는 여러가지 옵션이 있다.

대상 파일에서 문자열 검색:
grep 명령어에 문자열과 파일 이름을 지정하여 파일에서 문자열을 검색 가능.
ex) $ grep "찾고자 하는 문자열" FILE.txt -> FILE.txt에서 "찾고자 하는 문자열" 검색.

현재 디렉토리 모든 파일에서 문자열 검색:
파일 이름에 "*" 문자를 사용해서 현재 디렉토리에 있는 모든 파일을 대상으로 문자열을 검색 가능.
하지만 파일을 탐색할 뿐 현재 디렉토리에 있는 하위 디렉토리는 검사하지 않음.
만약 검사하기를 원한다면 -r 옵션을 이용해 하위 디렉토리에 있는 파일까지 검사 가능.
ex) $ grep "찾고자 하는 문자열" * -> 현재 디렉토리 모든 파일에서 "찾고자 하는 문자열" 검색.

특정 확장자를 가진 모든 파일에서 문자열 검색:
파일 이름 확장자 앞에 "*" 문자를 사용, 특정 확장자를 가진 모든 파일에서 문자열을 검색 가능.
ex) $ grep "찾고자 하는 문자열" *.txt -> txt 확장자를 가진 파일에서 "찾고자 하는 문자열" 검색.

대소문자 구분하지 않고 문자열 검색:
grep 명령어에 "-i" 옵션을 사용, 대소문자 구분없이 문자열 검색.
ex) $ grep -i "찾고자 하는 문자열" FILE.txt -> FILE.txt 파일에서 대소문자 구분없이 문자열 검색.

매칭되는 pattern이 존재하지 않는 라인 선택:
어떤 경우에는, 문자열이 매칭되는 라인이 아닌, 매칭되는 패턴이 존재하지 않은 라인을 선택해야 하는 경우가 있음.
이때 "-v" 옵션을 사용.
ex) $ grep -v "찾고자 하는 문자열" FILE.txt -> FILE.txt 파일에서 "찾고자 하는 문자열"이 포함되지 않은 라인을 표시.

단어 단위로 문자열 검색:
"-w" 옵션을 사용하면, 단어 단위로 문자열을 검색할 수 있습니다.
ex) $ grep -w "STRING" FILE.txt -> FILE.txt 파일에서 "STRING"이라는 문자열(단어 단위) 검색.

검색된 문자열이 포함된 라인 번호 출력:
"-n" 옵션을 사용하여, 검색 결과가 포함된 라인 번호를 출력.
ex) $ grep -n "찾고자 하는 문자열" FILE.txt -> "찾고자 하는 문자열"이 포함된 라인 번호 출력.

하위 디렉토리를 포함한 모든 파일에서 문자열 검색:
"-r" 옵션을 사용하면, 하위 디렉토리를 포함한 모든 파일에서 문자열을 검색 가능.
ex) $ grep -r "찾고자 하는 문자열" * -> "찾고자 하는 문자열"이 포함된 라인 번호 출력.

최대 검색 결과 갯수 제한:
grep 명령어의 결과가 너무 많이 표시될 때 "-m" 옵션 사용하여 최대 표시 결과 갯수 제한.
ex) $ grep -m 100 "찾고자 하는 문자열" FILE.txt -> FILE.txt 파일에서 "찾고자 하는 문자열"이 포함된 결과를 100개 까지만 표시.

검색 결과 앞에 파일 이름 표시:
"-H" 옵션을 사용하여 검색 결과 앞에 파일 이름을 표시.
ex) $ grep -H "찾고자 하는 문자열" * -> "찾고자 하는 문자열"이 포함된 파일 이름 표시.
ex) $ grep -Hn "찾고자 하는 문자열" * -> "찾고자 하는 문자열"이 포함된 파일 이름과 라인 번호 표시.

문자열 A로 시작하여 문자열 B로 끝나는 패턴 찾기:
정규 표현식에서 "."와 "*"를 조합하여 문자열 A로 시작하여 문자열 B로 끝나는 패턴 찾기 가능.
ex) $ grep "the.*step" * -> "the"로 시작하여 "step"으로 끝나는 패턴 검색.

ex) $ grep "A.*Z" * -> "A"로 시작하여 "Z"로 끝나는 패턴 검색.
[0-9] 사이 숫자만 변경되는 패턴 찾기:
정규 표현식 "[]"를 사용하여 0-9 사이 숫자만 변경되는 문자열 패턴을 검색 가능.
ex) $ grep step[0-9] * -> "step0", "step1", ..., "step9" 패턴을 검색.

문자열 패턴 전체를 정규 표현식 메타 문자가 아닌 일반 문자로 검색하기:
"-f" 옵션을 사용하면, 패턴에 지정된 문자열을 메타 문자로 인식하지 않고 일반 문자로 인식하여 패턴 검색.
ex) $ grep -f "[0-9]" -> "[0-9]" 문자열 검색.

정규 표현식 메타 문자를 일반 문자로 검색하기:
문자열 패턴에서 정규 표현식 메타 문자 앞에 "\"를 사용하면, 해당 문자를 일반 문자로 인식하게 할 수 있음.
ex) $ grep "\*" FILE.txt -> FILE.txt 파일에서 * 문자 검색.
ex) $ grep "\." FILE.txt -> FILE.txt 파일에서 . 문자 검색.

문자열 라인의 처음 시작 패턴 검색하기:
정규 표현식 "^"를 사용하여 문자열 라인의 중간이 아닌 시작 패턴만 검색 가능.
ex) $ grep "^C" FILE.txt -> FILE.txt에서 "C"로 시작하는 라인 검색.
ex) $ grep "^1" FILE.txt -> FILE.txt에서 "1"로 시작하는 라인 검색.

문자열 라인 마지막 종료 패턴 검색하기:
정규 표현식 "$"를 사용, 문자열 라인의 처음 또는 중간이 아닌 종료 패턴을 검색 가능.
ex) $ grep "\.$" FILE.txt -> FILE.txt "." 으로 끝나는 라인 검색.
ex) $ grep -v "\.$" FILE.txt -> FILE.txt "."으로 끝나지 않는 라인 검색.
